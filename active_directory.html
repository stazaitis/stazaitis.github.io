Building a Directory Service Server with Active Directory

Overview

This project demonstrates setting up an enterprise-level Active Directory domain environment.

The Directory Services Server functions as the central authentication and management hub, operating through three core components:

DNS (Domain Name System)

When one of my desktop machines attempts to access google.com, the request is first sent to the Directory Services Server. This server processes the request by delegating the query and retrieving the corresponding IP address for google.com, allowing seamless domain resolution within the enterprise network.

DHCP (Dynamic Host Configuration Protocol)

The DHCP server dynamically assigns IP addresses to workstations across the enterprise network, ensuring efficient IP management and connectivity without requiring manual configuration.

Identity & Single Sign-On (SSO)

The Active Directory service centrally manages user accounts, computer identities, and authentication policies across the enterprise network. It maps users to their designated workstations, including Ubuntu and Windows 11 Enterprise machines.

With SSO (Single Sign-On), users can authenticate once and gain secure access to multiple enterprise applications without repeated logins.
